# Security Policy

## Supported Versions

The maintainers of **pabot** aim to support the latest released version.

Security updates will generally not be provided for older versions.

---

## Reporting a Vulnerability

If you discover a security vulnerability in **pabot**, please **do not open a public issue**.

Instead, report it privately so we can address it responsibly.

### How to report

Please contact the maintainers via one of the following options:

- GitHub Security Advisories (preferred, if enabled)
- Or by contacting the maintainers directly (see repository maintainers)

When reporting, please include:

- A description of the vulnerability
- Steps to reproduce (if applicable)
- Potential impact
- Any suggested mitigation or fix

---

## Disclosure Process

Once a vulnerability is reported:

1. The maintainers will acknowledge the report
2. The issue will be investigated
3. A fix will be prepared and released
4. A public disclosure may be made after a fix is available

We appreciate responsible disclosure and will do our best to respond promptly.

---

Thank you for helping keep **pabot** and its users secure.
